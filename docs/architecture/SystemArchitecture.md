# DeveloperOS V1 系统整体架构

## 一、总体架构

```text
┌─────────────────────────────────────────────┐
│                  Client                     │
│            Web（Vue3 + TypeScript）          │
└─────────────────┬───────────────────────────┘
                  │ HTTP / SSE
                  ▼
┌─────────────────────────────────────────────┐
│               Spring Boot API               │
│                                             │
│  ┌─────────────┐  ┌──────────────────────┐  │
│  │ Auth Module │  │  User Module         │  │
│  └─────────────┘  └──────────────────────┘  │
│                                             │
│  ┌─────────────┐  ┌──────────────────────┐  │
│  │ Chat Module │  │  Memory Module       │  │
│  └─────────────┘  └──────────────────────┘  │
│                                             │
│  ┌─────────────┐  ┌──────────────────────┐  │
│  │ RAG Module  │  │  Growth Module       │  │
│  └─────────────┘  └──────────────────────┘  │
└─────────────────┬───────────────────────────┘
                  │
        ┌─────────┼──────────┬──────────────┐
        │         │          │              │
        ▼         ▼          ▼              ▼
┌──────────┐ ┌─────────┐ ┌─────────┐ ┌────────────┐
│   LLM    │ │ Redis   │ │ PostgreSQL │ │ pgvector │
│ OpenAI等 │ │ 缓存记忆 │ │ 业务数据   │ │ 向量检索 │
└──────────┘ └─────────┘ └─────────┘ └────────────┘
```

------

# 二、模块职责

## Auth Module（认证模块）

负责：

- 登录
- 注册
- JWT 鉴权
- 权限校验

------

## User Module（用户模块）

负责：

- 用户信息
- 用户设置
- 用户偏好

------

## Chat Module（AI 对话模块）

负责：

- 会话管理
- 消息管理
- SSE 流式输出
- Prompt 组装
- 多轮上下文

------

## Memory Module（长期记忆模块）

负责：

- 用户长期记忆
- 用户画像
- 记忆提取
- 记忆检索
- 重要信息沉淀

------

## RAG Module（知识库模块）

负责：

- 文件上传
- 文档切片
- Embedding
- 向量存储
- 检索增强问答

------

## Growth Module（成长模块）

负责：

- 学习目标
- 学习任务
- 成长日志
- 成长记录

------

# 三、数据存储职责

## PostgreSQL

存储：

- 用户
- 聊天记录
- 学习计划
- 成长日志
- 文档信息
- 长期记忆

------

## Redis

存储：

- Token
- 热点缓存
- 会话上下文缓存
- 临时记忆

------

## pgvector

存储：

- 文档向量
- 知识片段向量
- 长期记忆向量（后续）

------

# 四、外部依赖

## 大模型服务

支持：

- OpenAI
- Claude
- Gemini
- 国产模型（后续）

------

# 五、架构特点

✅ 单体架构，开发简单

✅ 模块化设计，方便后续拆分

✅ 支持 AI 与 RAG 场景

✅ 支持长期记忆能力扩展

✅ 后续可平滑升级为微服务
